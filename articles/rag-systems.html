📌 Introduction

Retrieval-Augmented Generation (RAG) supercharges AI language models by giving them access to your own data at query time. Instead of relying solely on what the model learned during training, RAG lets it search a knowledge base and use that information to produce accurate, grounded answers.

💡 Who This Is ForThis is a Advanced-level article. Estimated reading time: 15 min.
Advertisement

💡 Key Concepts

Vector Embeddings

Text is converted into lists of numbers (vectors) capturing semantic meaning. Similar concepts cluster near each other in this mathematical space.

Vector Database

A specialised database (Pinecone, Weaviate, pgvector) that stores embeddings and enables ultra-fast similarity search.

Chunking

Breaking documents into smaller overlapping pieces before embedding so the retriever can find the most relevant passage.

Retriever + Generator

The retriever finds relevant chunks; the generator (LLM) reads them and writes a grounded response.

📊 Code Examples

python
from openai import OpenAI
from pinecone import Pinecone, ServerlessSpec

client = OpenAI()
pc = Pinecone(api_key="YOUR_API_KEY")

pc.create_index(
    name="knowledge-base", dimension=1536, metric="cosine",
    spec=ServerlessSpec(cloud="aws", region="us-east-1")
)
index = pc.Index("knowledge-base")

def embed_text(text: str) -> list[float]:
    resp = client.embeddings.create(model="text-embedding-3-small", input=text)
    return resp.data[0].embedding

def upsert_documents(docs: list[dict]):
    vectors = [{
        "id": doc["id"],
        "values": embed_text(doc["text"]),
        "metadata": {"text": doc["text"], "source": doc["source"]}
    } for doc in docs]
    index.upsert(vectors=vectors)
python
def rag_query(question: str, top_k: int = 5) -> str:
    # 1. Embed the question
    q_vec = embed_text(question)
    # 2. Retrieve relevant chunks
    results = index.query(vector=q_vec, top_k=top_k, include_metadata=True)
    # 3. Build context
    context = "\n\n".join(
        f"Source: {r.metadata['source']}\n{r.metadata['text']}"
        for r in results.matches
    )
    # 4. Generate grounded answer
    resp = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role":"system","content":"Answer using only the provided context. If unsure, say so."},
            {"role":"user","content":f"Context:\n{context}\n\nQuestion: {question}"}
        ]
    )
    return resp.choices[0].message.content

print(rag_query("What is our refund policy?"))

⚠️ Common Mistakes

⚠️ Chunk size too large
Chunks over 512 tokens dilute retrieval precision. Use 256–512 tokens with 10–20% overlap.
⚠️ No metadata filtering
Without metadata filters (date, source, department), retrieval becomes noisy. Attach and filter on metadata.
⚠️ Skipping re-ranking
Top-K vector search isn't always perfect. Add a cross-encoder re-ranker to improve final result quality.

✅ Best Practices

  • ✅ Use overlapping chunks (50–100 token overlap) to avoid splitting key context
  • ✅ Store source metadata with every chunk for traceability
  • ✅ Implement hybrid search: combine vector similarity + BM25 keyword search
  • ✅ Cache embeddings to avoid redundant API calls on unchanged content
  • ✅ Monitor retrieval quality with recall@K metrics on a test question set

🚀 Real-World Use Cases

🚀 Customer Support Bot

Index your entire help center. The agent retrieves relevant articles and writes clear answers with source citations.

🚀 Internal Knowledge Base

Give employees chat access over Confluence or Notion. HR policies and runbooks become instantly searchable.

🚀 Legal Document Analysis

Ingest contracts and regulations. Ask 'What are the termination clauses?' and get accurate cited answers.

🔧 Tools & Resources

Pinecone

Managed vector database. Easy setup, fast queries, production-ready.

pgvector

Open-source vector extension for PostgreSQL. One database for everything.

📌 Summary

  • RAG combines retrieval with generation to produce grounded, accurate answers
  • Chunking strategy directly affects retrieval quality — test empirically
  • Use metadata filtering, hybrid search, and re-ranking for production-grade precision
  • Monitor retrieval quality continuously with test sets and feedback loops
